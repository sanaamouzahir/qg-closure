## 5. diagnose_error_distribution.py — the answer

INIT model, FRC-256 @ 5e-3, val (n=225):
  Ndot   mean=0.182  median=0.186
  Nddot  mean=1.177  median=0.265
  N3dot  mean=51.68  median=0.331   p90=256  max=711
Worst offenders: windows 30/34/40/51..., ||tN3||≈1.2e2 (vs healthy 7.3e6),
stack Delta^2 ≈ 7e-7 (FROZEN).

TRAINED model (best.pt), same split:
  N3dot median=0.996 EVERYWHERE (model zeroed the head);
  Ndot/Nddot medians WORSE than init (0.27/0.31 vs 0.19/0.26);
  its "best" samples are the quiescent ones (pred≈0 fits tiny targets).

DIAGNOSIS: early windows are quasi-zonal spin-up flow. Zonal state =>
J(psi,omega)≈0 identically => true N-derivatives legitimately tiny (1e2 vs
7e6). FD on the frozen stack returns float32 noise /dt^3 ≈ 2e4. Per-sample
RELATIVE error = 2e4/1.2e2 ≈ 200-700 ✓ matches observed. ~9-33% of windows
per member (rises with beta: combo 9%, FRC-256 16%, b2 27%, b25 33% — high-beta
members stay zonal longer). These samples poisoned TRAINING: optimizer's
cheapest move is shrinking predictions toward zero -> destroyed mix, N3dot=0.99,
trained-worse-than-init.

t-start=15 was calibrated on low-beta members; insufficient for high-beta.

## 6. Fix applied

filter_quiescent_windows.py: drop whole windows with
  median-over-anchors ||tN3|| < 1e-2 x member median, OR stack roughness < 1e-5.
Backed up splits to split_prefilter.npz. Dry-run drops (identical across the
3 dts per member — targets dt-independent ✓):
  FRC-256 79/500 (15.8%), b2 134/500 (26.8%), b25 166/500 (33.2%),
  combo 45/500 (9.0%), ...

Retrain: from PHYSICS INIT (not best.pt), lr 5e-5, filtered splits, all
members kept (Re25k's "catastrophe" was this same artifact — no member cut).

## 7. Rules extracted (in CLAUDE.md)

- Spin-up duration scales with beta: t-start must be per-member (zonal phase);
  filter windows by target norm + stack roughness before training.
- Per-sample relative metrics explode on near-zero-denominator samples:
  report MEDIANS alongside means; never train unfiltered.
- Trained-worse-than-init = poisoned-pool signature; the distribution probe
  (median vs mean + worst-offender table) is the diagnostic.
- Debug order that worked: per-root eval -> init eval -> mark smoothness ->
  sliced-vs-deep byte compare -> single-sample stage audit -> distribution.
